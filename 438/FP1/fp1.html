FP1: Hoops Prototype

Overview & Usage

For this project, I've made a game using p5js and p5play. The game is a basketball hoop shooter, where the user can click and drag the mouse to shoot a ball into a basket. This final prototype builds on my MP3 game by adding a scoreboard with a Firestore database.

The game can be played here:

Hoops Prototype

Development Process

I started this project by setting up a new repository and importing my code from my MP3 game. Next, I created a back-end database with Firebase to store players' high scores. I tested sending scores to the database at the end of a game.

I ran into some issues starting a new repository and getting the game to send and receive data to Firebase. Once those were sorted out, I formatted the scoreboard as a basic text display once the timer runs out. This way, the player can keep shooting endless baskets after the game ends.

The last step for the prototype deliverable on 3/7 was to create a start screen where the user can input their name and press a button to start the game. Once this overlay was implemented, I decided on a color palette for the game and started to put it in.

Issue Deep-Dive

My biggest issue was properly connecting the Firestore database to the game. First, I had issues properly importing and setting up firebase dependencies in the code. This was because my code-bundling setup with vite wasn't configured properly, and I couldn't quite figure out how to set the internal rollup configuration within vite. I was trying to fix a logic error where files weren't being imported properly, but every change I made caused a syntax error. As a work-around, I ended up installing rollup in the repository to easily define which files should be included in the dist/ directory. I learned that rollout and vite don't conflict with each other if you install both, and it ended up being easiest to install both this time. Once that was figured out, I had to fix a few minor syntax errors in my code to send and receive data from the database. Now, it all works to save and display players' names, scores, and shots taken.
